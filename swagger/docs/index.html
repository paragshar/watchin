Documentation

The webservice is used for managing users, events, posts and comments.

API-URL

https://someurl.com/ Be aware of the https

API-Key

The api key has to be added as a GET parameter called 'apiKey' added to each request.

Authentication

The authentication is done through access tokens. Whenever the POST /user/authenticate method is called, a new access token for the specified user is generated. This token has to be passed as an additional 'Access-Token' request header to each request that needs authentication.

Request headers

Accept: application/json
Content-Type: application/x-www-form-urlencoded

Request body

If not specified differently, the body has to be sent as form data.

JSONP

JSONP is now supported by adding the GET-Parameter 'jsonp=functionName'

Response codes

The following are global response codes, which can be returned at any request. Request-specific status codes are listed at each operation.

  • 200 Success
  • 400 Required fields are missing
  • 401 Permission denied
  • 403 Forbidden (action not allowed for this object)
  • 405 Method not allowed (HTTP method not allowed for this request)
  • 409 Conflict (no object by selected parameter or unique constraint fails.)
  • 415 Invalid data type
  • 500 Fatal server error
